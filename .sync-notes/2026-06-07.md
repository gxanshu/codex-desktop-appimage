no `rustc`). Without `g++` the native-modules rebuild step cannot run,
and without `cargo` the `computer-use-linux` / `read-aloud-linux`
binaries cannot be rebuilt locally.

Verifications that **did** pass locally:

- `node --test scripts/patch-linux-window-ui.test.js` — **193 tests passed**.
- `node --test linux-features/*/test.js` — every feature suite passed
  except `example-feature` and `read-aloud-mcp`, both of which fail
  identically: their child processes spawn bare `node` and the sandbox
  has no `node` on the default `PATH`. These are environment issues, not
  patch regressions.

The CI workflow (`rust-and-smoke`) is the gating job and will run the
full suite on GitHub-hosted Ubuntu with `rustc`, `cargo`, `g++`, and
Node 24 on `PATH`. Merging this PR is contingent on that job going green.

This PR is **draft** until either (a) the runner gets a build-capable
toolbox, or (b) GitHub Actions CI confirms the merge resolution is
correct end-to-end.
